# 圆环工作流设计

当前建议先把圆环作为第一条标准几何工作流打通，主链如下：

`Plane Selector / Construct Plane / Construct Plane From Points`
-> `Torus (Blocks)`
-> `Block Type Selector` 或 `Material Mapper`
-> `Geometry Viewer`
-> `Apply Changes`

## 最小可用链

1. `Plane Selector`
输出标准平面，适合快速在 `XY / YZ / XZ` 三个常见平面上放置圆环。

2. `Construct Plane`
用 `Origin + Normal` 生成平面，适合和其他向量节点组合。

3. `Construct Plane From Points`
用三个非共线点生成平面，适合根据场景中的已知点位构造斜平面。

4. `Torus (Blocks)`
输入：
- `Center`
- `Plane`，可选
- `Major Radius`
- `Minor Radius`

行为：
- 未连接 `Plane` 时，默认使用世界 `Y` 轴方向。
- 连接 `Plane` 后，按平面法线确定圆环朝向。
- 未连接 `Center` 但连接了 `Plane` 时，默认使用平面原点作为圆环中心。

5. `Block Type Selector` 或 `Material Mapper`
- 单一材质时优先使用 `Block Type Selector`
- 需要做分层或规则映射时使用 `Material Mapper`

6. `Geometry Viewer`
负责预览几何结果，确认造型后再进入实际写入。

7. `Apply Changes`
负责把结果正式写入 Minecraft 世界。

## 当前里程碑

- 执行链路已经开始向稳定协议收敛，节点输出会同步到输出端口。
- 圆环节点已支持可选 `Plane` 输入。
- 已补齐基础平面来源节点：
  - `Plane Selector`
  - `Construct Plane`
  - `Construct Plane From Points`

## 下一步建议

- 增加“从玩家朝向构造平面”节点。
- 增加“从选中方块表面法线构造平面”节点。
- 给圆环节点补充厚度、空心、局部截取等参数。
- 提供一份可以直接加载的示例图：
  - 地面圆环
  - 竖直圆环
  - 倾斜圆环
